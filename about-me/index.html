Software Technologies

The following is a list of the software technologies that I have been learning over the last while:

  • React: A component based Javascript/Typescript framework to build modern single-page web applications.
    • GSAP: A Javascript framework (with React hooks) for building animations in the browser. I gravitated towards this framework because it's timeline features are similar to keyframe oriented animation in Adobe products like After Effects and Premiere Pro.
    • Framer Motion: Another Javascript framework for building animations. I no longer use this framework and have completely converted to GSAP because GSAP has a more inuitive and flexible development experience.
  • Tensorflow: A Python machine learning and artificial intelligence library which is useful for building and training Neural Networks of all kinds.
    • Keras: A Tensorflow library for creating Neural Networks for Deep Learning. Keras makes it even more simple to create Neural Networks by providing a simple API to interface with Tensorflow. It has many out-of-the-box features for creating both simple and complex models, and useful uilities for preprocessing data.
    • OpenCV: Another AI library, however this library is a Computer Vision specific (a subfield of Artificial Intelligence that aims at making computers able to see). OpenCV makes it easier to create Computer Vision models of all kinds.
  • Continuous Integration / Continuous Delivery (CI/CD): I have used GitHub Actions for automating the delivery of some of my projects.
  • Docker: A containerization technology that allows developers to create ( and most importantly to recreate ) lightweight virtualized environments at the operating system level, either locally or remotely.
  • Firebase: Firebase is a Google-provided PaaS that has a host of services such as databases and OAuth solutions, that are easily implemented with the Firebase SDK. I used Firebase for the OAuth and login features of CharlemagneDB and found the documentation to be very good too.

Content Creation Technologies

The following is a list of the content creation technologies that I have used:

  • Adobe Photoshop: This is software for editing images but more specifically raster graphics (pixelated graphics). I used this software to edit images across my content creation projects and web projects such as thumbnails and logos.
  • Adobe After Effects: This software is used for advanced video effects that are not possible on Premiere Pro such as tracking with null objects. After Effects is used for detailed short clips that require a lot of hardware intense rendering, and comes with plugins such as Geolayers 3, that I used for map animations in my videos.
  • Adobe Premiere Pro: I used this software for long form video editing, and in contrast to After Effects. I would use this to get basic tasks done fast, such as cuts, audio editing and transitions.
  • Adobe Illustrator: I used Illustrator to create vector graphics such as .svg and .ai format images, which (in contrast to rasterized images) have perfect quality at all scales. Illustrator provides more tools for working with vector shapes than Photoshop such as the shape builder tool.
  • Adobe XD: I started out designing my UX designs in Adobe XD, I designed the CharlemagneDB and created scroll animations and layouts for both mobile and PC versions of the website.
  • Figma: When my computer had to be repaired I decided to migrate to Figma for UX designs and ended up preferring it over XD.
